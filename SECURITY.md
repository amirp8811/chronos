# Security Policy & Vulnerability Disclosure

The CHRONOS architecture implements clean-slate anonymous routing and post-quantum cryptographic primitives (ML-KEM-768, X25519, ChaCha20-Poly1305, and Sphinx-PQC encapsulation). We treat security, memory safety, and timing side channels with the absolute highest priority.

## Reporting a Security Vulnerability
Please DO NOT open a public GitHub Issue for critical vulnerabilities, cryptographic side-channel leaks, or directory BFT consensus exploits.
* Email your report directly to: amirp8811@gmail.com
* Subject Line: [CHRONOS SECURITY DISCLOSURE] - Brief Summary
* Response Time: You will receive an initial acknowledgment from the maintainers within 24 hours.
